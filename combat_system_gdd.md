# 战斗系统 GDD v0.3：D6 构筑、Run 成长与三部位装备

## 1. 核心定位
本项目的战斗核心是“多颗标准六面骰构筑”。角色不是携带技能牌组，而是携带多颗 D6；每颗 D6 有 6 个面，每个面代表概率、风险、节奏和战斗风格。

第一阶段以单角色深挖为主：每名角色初始携带 3 颗 D6，每回合全掷全用。构筑深度来自新增骰子、替换骰面、升级骰面、附魔、调整结算顺序、管理负面面、使用装备、利用角色资源和应对敌人意图。

目标参考《杀戮尖塔》的路线构筑深度与 Astrea 的骰子风险构筑，但表达必须服务本项目自己的角色风格、局内成长和局外养成。

## 2. 回合规则
每场战斗由玩家角色对抗一个或多个敌人。MVP 先做单角色对单敌人，底层为多角色、多敌人预留。

每回合流程：
1. 敌人展示本回合意图。
2. 玩家掷出当前装备的全部骰子，起始为 3 颗 D6。
3. 每颗骰子产生 1 个面，默认本回合全部必须结算。
4. 玩家可通过重掷、锁定、延迟、换序、净化、强化、装备主动道具等方式改变结果或结算。
5. 骰子按玩家选择顺序结算。
6. 敌人执行意图。
7. 回合结束，处理持续效果、资源衰减、负面代价、装备回合钩子和胜负判定。

默认每回合有 2 次基础重掷。额外操作主要来自骰面、装备、升级、局内奖励和角色机制，而不是固定 UI 按钮堆叠。

## 3. 骰子结构
每颗骰子必须是标准 D6，拥有 6 个面。骰子是构筑单位，骰面是概率单位。

骰子类型：
- `Stable Die`：稳定骰，负面少，收益平滑，适合保底和新手。
- `Volatile Die`：风险骰，强面和负面面并存，适合爆发构筑。
- `Engine Die`：引擎骰，直接数值低，但提供资源、改骰、召唤、标记或顺序收益。
- `Finisher Die`：终结骰，依赖资源、状态或条件，负责构筑收束。
- `Support Die`：支援骰，来自召唤物、无人机、战旗、装备或局内奖励。

骰面标签：
`attack`、`defense`、`resource`、`setup`、`finisher`、`control`、`summon`、`negative`、`cleanse`、`corrupt`、`counter`、`mark`、`pierce`、`delay`。

负面面不是单纯惩罚，而是构筑材料。它可以造成自伤、过热、失去格挡、给敌人护甲、污染下一颗骰子、强制延迟结算、消耗资源或触发敌人反制。风险骰必须让玩家产生“我要不要拥抱这个危险”的判断，而不是被动接受随机惩罚。

## 4. 数值基准
普通战目标长度为 6-9 回合。低于 5 回合说明爆发过强，高于 10 回合说明拖沓。

玩家基础数值：
- 普通攻击面：3-6 伤害。
- 强攻击面：7-10 伤害，但需要条件、资源或负面代价。
- 普通防御面：3-6 格挡。
- 强防御面：7-10 格挡，但通常牺牲输出、带延迟或消耗资源。
- 资源面：获得 1-2 点角色资源。
- 引擎面：即时数值偏低，但提供改骰、召唤、标记或下回合收益。
- 负面面：代价约等价于 2-5 点生命、格挡或节奏损失。

敌人基础数值：
- 普通敌 HP：45-65。
- 精英 HP：75-110。
- Boss HP：140-220，按阶段拆分压力。
- 普通攻击：4-8。
- 强攻击：9-14，需要提前展示。
- 蓄力攻击：14-22，必须提供应对窗口。

## 5. 角色设计与具体 BD
角色风格必须统一，但局内构筑分支可以多样。

### 5.1 Cyan Ryder：高科技敏捷
关键词：过载、闪避、多段、冷却、无人机、改骰。主资源：`overload`。

基础三骰：`Pulse Die`、`Shift Die`、`Core Die`。

BD A：过载爆发
- 核心：快速获得 overload，再把 overload 转化为多段爆发或终结伤害。
- 关键面：获得 overload、多段攻击、消耗 overload 强化下一次攻击、过热负面。
- 风险：overload 溢出会自伤、锁骰或降低下回合稳定性。
- 成型标志：一回合内能主动积累 3-5 overload，并把它转成爆发窗口。

BD B：闪避反击
- 核心：根据敌人意图选择闪避、防御、反击和换序。
- 关键面：闪避、延迟结算、反击、重掷和锁定。
- 风险：敌人不攻击时收益空转，需要补输出或装备支持。
- 成型标志：强攻敌人反而成为 Cyan 的输出窗口。

BD C：无人机连射
- 核心：召唤 drone，利用多段、mark 和装备让无人机成为独立输出轴。
- 关键面：召唤、无人机攻击、冷却、信号标记。
- 风险：反召唤敌人会清无人机或污染 summon 面。
- 成型标志：无人机不只是额外伤害，而是能触发和承接构筑链。

### 5.2 Helios Windchaser：游侠
关键词：标记、追踪、陷阱、召唤伙伴、精准终结、箭袋。主资源：`quiver`。

基础三骰：`Hunt Die`、`Mark Die`、`Wild Die`。

BD A：印记狙杀
- 核心：铺 mark、保 mark、爆 mark。
- 关键面：施加 mark、消耗 mark 造成穿透、精准终结。
- 风险：误导标记会给错误目标或触发敌人反制。
- 成型标志：玩家能围绕目标身上的 mark 层数规划击杀窗口。

BD B：伙伴召唤
- 核心：召唤伙伴并通过 mark、防御和装备触发伙伴追加攻击。
- 关键面：召唤、伙伴攻击、伙伴护卫、伙伴失控负面。
- 风险：伙伴被清除或失控时会消耗 quiver 或浪费节奏。
- 成型标志：伙伴成为一条稳定副轴，而不是随机附赠。

BD C：陷阱箭雨
- 核心：提前布置 trap，延迟造成伤害并配合 mark 触发箭雨。
- 关键面：陷阱、延迟伤害、群体打击、箭袋消耗。
- 风险：陷阱延迟导致短期防线薄弱。
- 成型标志：玩家能利用敌人意图窗口安排延迟爆发。

### 5.3 Aurian：战士
关键词：站场、武器、格挡、防反、蓄力、处决、怒气。主资源：`stance`。

基础三骰：`Blade Die`、`Guard Die`、`Might Die`。

BD A：防反战士
- 核心：通过高 block 和 counter 把敌人攻击转成输出。
- 关键面：格挡、防反、嘲讽、反击。
- 风险：破防会清空反击节奏或损失 stance。
- 成型标志：敌人强攻越多，Aurian 的输出窗口越大。

BD B：蓄力处决
- 核心：用 charge 和 stance 准备大回合，再用 execute 收束。
- 关键面：蓄力、强攻击、延迟、处决。
- 风险：蓄力期间防御不足，反延迟敌人会施压。
- 成型标志：玩家围绕“大回合”规划，而不是每回合平均输出。

BD C：狂战投掷
- 核心：自伤、风险面和投掷攻击互相转化。
- 关键面：自伤换伤害、穿透、投掷、怒气失控。
- 风险：血线管理压力高。
- 成型标志：玩家主动拥抱负面面，但有装备和骰面把风险变成节奏。

## 6. 跨角色联动口子
当前阶段仍是单角色战斗，但系统必须为未来多角色和支援角色保留口子。

共享标签：
- `mark`：Helios 主用，Cyan 无人机和 Aurian 处决也能读取。
- `counter`：Aurian 主用，Cyan 闪避反击可读取。
- `summon`：Helios 伙伴和 Cyan 无人机共用。
- `delay`：Helios 陷阱、Aurian 蓄力、Cyan 冷却都可用。
- `pierce`：Helios 精准、Aurian 投掷、Cyan 高科技武器都能使用。

原则：
- 每个角色至少一条 BD 能读取其他角色的核心标签。
- 跨角色联动优先通过装备、附魔、通用骰子和事件奖励实现。
- 不要求所有 BD 互通，避免角色风格被稀释。

## 7. Run 路线与成长曲线
默认一局 Run 为 13 层：前期 1-4，中期 5-8，深层 9-12，第 13 层 Boss。玩家每层从当前可达节点中选择 1 个推进。

成长曲线：
- 1-4 层：起手塑形，确定第一条方向。奖励以新 D6、替换骰面、基础装备为主。
- 5-8 层：引擎成型，加入商店、精英、强事件、附魔和流派奖励。
- 9-12 层：收束压力，敌人强度明显上升，奖励更少但更关键。
- 13 层：Boss，检验构筑闭环。

撤离窗口：
- 清完第 4 层可撤离：保守止盈。
- 清完第 8 层可撤离：中期成型判断。
- 清完第 12 层可撤离：Boss 前高压选择。
- 有 pending reward 时不能撤离，避免跳过奖励结算。

节点类型：
- `battle`：常规构筑验证。
- `elite`：高压构筑检验，奖励更强。
- `event`：提供风险选择和构筑分叉。
- `supply`：低风险修补、回复、小型构筑补强。
- `shop`：定向购买骰子、装备、附魔、修理。
- `rest`：治疗、修理装备、稳定升级。
- `boss`：最终构筑检验。

## 8. 局内奖励
局内成长必须显著高于局外成长，并且优先改变构筑，而不是只加小数值。

奖励优先级：
1. 新增 D6。
2. 替换骰面或替换整颗 D6。
3. 骰面附魔。
4. 获得装备。
5. 修理装备。
6. 净化负面面。
7. 角色专属机制奖励。
8. 少量 HP 或 credits 作为副奖励。

当前支持的奖励类型：
- `add_die`
- `replace_die`
- `remove_negative`
- `upgrade_die`
- `equipment`
- `growth`
- `currency`

## 9. 骰面附魔系统
附魔作用于单个骰面，是局内构筑的第二层深度。

基础规则：
- 每个骰面最多 1 个附魔。
- 附魔可替换，不可叠加。
- 附魔可作用于负面面。
- 强附魔主要来自精英、锻造、深层事件和 Boss 奖励。

首批附魔设计：
- `Double Pips`：该面数值翻倍，负面也翻倍。
- `Echo`：该面结算两次，第二次数值减半。
- `Clean Edge`：移除 `negative`，数值 -1。
- `Risk Edge`：数值 +3，追加轻负面。
- `Delay Spark`：延迟到回合末，数值 +50%。
- `Piercing Rune`：攻击面获得 `pierce`。
- `Marking Rune`：攻击面追加 mark 1。
- `Guard Rune`：非防御面结算后获得 block 2。
- `Battery Rune`：资源面额外 +1，下回合首个骰面 -1。
- `Replay Rune`：每场战斗第一次掷到该面时复制一次结果。

## 10. 三部位装备系统
装备是局外长期记忆与局内 BD 启动器。每名角色固定三部位：`weapon`、`armor`、`item`。装备跨角色通用，但不能跨部位装备。

### 10.1 部位定位
- `weapon`：输出、资源启动、流派触发、终结强化。
- `armor`：生存、防反、负面缓冲、召唤保护、风险兜底。
- `item`：混合槽，可放被动饰品或主动道具，负责改骰、事件收益、附魔和临场爆发。

硬规则：
- 每个角色最多装备 1 件武器、1 件护甲、1 件道具。
- 武器不能装到护甲或道具槽。
- 护甲不能装到武器或道具槽。
- 道具不能装到武器或护甲槽。
- 装备没有角色限制，强适配来自标签联动。

### 10.2 带回与损坏
- Run 中获得装备时立即装备到对应部位。
- 如果该部位已有装备，家中带出的旧装备进入临时背包；本 Run 新获得的旧装备默认丢弃。
- 撤离或通关后，装备槽和临时背包中的装备进入仓库。
- 战败时，新获得装备丢失；家中带出的装备返回。
- 装备记录 `return_count`。
- 第 1 次带回安全。
- 第 2 次带回开始损坏：25%。
- 第 3 次：40%。
- 第 4 次及以后：55%，上限 70%。
- `intact -> damaged -> broken`。
- `broken` 不能出发，只能修理或分解。

仓库规则：
- 初始仓库容量 8。
- 局外成长每级 +4。
- MVP 上限 24。
- 仓库满时当前原型先自动分解最低优先级装备，后续改成结算整理 UI。

### 10.3 首批装备
武器：
- `Pulse Carbine`：首次多段攻击后获得资源，服务 Cyan 过载、Helios 箭雨、Aurian 投掷。
- `Hunter Scope`：首次 mark 增强；没有 mark 能力的角色首次攻击追加 mark。
- `Blood Whetstone`：每回合首次自伤或负面后强化下一次攻击。
- `Glass Fang`：攻击固定增伤，但偏风险 BD。
- `Throwing Harness`：首次 finisher 获得穿透和资源返还。

护甲：
- `Heat Sink Plating`：首次 overheat 或自伤负面改为 block 损失。
- `Iron Banner`：首次单回合获得 8+ block 时获得 counter/thorns。
- `Guardian Totem`：有召唤物时首次受到攻击减伤。
- `Field Jacket`：战斗开始获得 block。
- `Spiked Guard`：有 block 时被攻击前反击；未被攻击会损失少量 block。

道具：
- `Lucky Knuckle`：每场第一次重掷免费。
- `Old Compass`：事件节点出现额外安全选项。
- `Micro Drone Dock`：主动，每场一次，召唤临时 drone 并造成小伤害。
- `Companion Whistle`：主动，每场一次，命令伙伴攻击，目标有 mark 时追加伤害。
- `Forge Needle`：偏附魔路线的被动道具。
- `Risk Ledger`：每触发 2 次负面面获得重掷。
- `Emergency Battery`：主动，每场一次，获得资源，下回合首个攻击轻微受损。

## 11. 敌人与挑战
怪物的职责是提供可读挑战，而不是随机惩罚玩家。

敌人意图类型：
`attack`、`block`、`charge`、`debuff`、`summon`、`cleanse_self`、`counter_tag`、`pollute_die`、`lock_die`、`phase_shift`。

普通怪用于教学不同问题：
- 高攻击怪考验防御、闪避和反击。
- 护甲怪考验破甲、穿透和持续收益。
- 召唤怪考验优先级。
- 污染怪考验净化与重掷。
- 反连段怪考验结算顺序。

精英怪检验构筑完整性：反多段、反防御、反召唤、反蓄力，但每种反制都必须提前展示，并至少有两种应对方式。

Boss 三阶段：
- P1 展示规则。
- P2 限制玩家惯用解法。
- P3 给倒计时或强压迫，迫使玩家用构筑完成收束。

## 13. 测试与平衡标准
数据测试：
- 每个 `die_id` 必须刚好 6 个面。
- 每名初始角色必须刚好 3 颗 D6。
- 每个装备必须有合法 `equip_slot`。
- 主动道具必须声明 `item_mode=active`。
- `broken` 装备不能出发。

Run 测试：
- 第 4、8、12 层后可撤离。
- 有 pending reward 时不能撤离。
- 撤离/通关后装备进入仓库。
- 战败不带回新装备。
- 家中带出的装备战败后返回。
- 第二次带回开始触发损坏判定。
- 任意角色都能装备任意合法部位装备。

战斗测试：
- Cyan 能稳定触发 overload、冷却或无人机。
- Helios 能稳定触发 mark、召唤或陷阱。
- Aurian 能稳定触发 counter、蓄力或处决。
- 装备能改变战斗行为，而不是只增加最终数值。
- 主动道具按钮只在 item 槽装备主动道具时出现。
- damaged 装备效果弱于 intact。

平衡测试：
- 每角色对每个普通怪跑 100 局。
